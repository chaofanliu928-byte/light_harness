核心问题:老版本是怎么失败的?

场景:旧版本(只有规则文本,没有"保安")
👤 小李想改 governance/design-rules.md 加一条新规则
📜 现有规则写着"改这种文件前要走专家审查"
🤔 小李心想
规则写在那儿,但又没人盯着...算了,直接改吧。
✏️ 行动直接改 + commit + 干别的去了
📊 实际数据:9 天 / 14 session / 142 commit / 16 spec — 专家审查触发 0 次。

问题诊断:规则只是"写"在那儿,没机制"拦"。AI 自律 → 不靠谱。

P0.9.1 的修法:派两个保安站门口

把"是否走审查"从"AI 自己决定"改成"门口有保安拦,过不去的就得交代清楚"。

场景 A:走正路 — 开会留纪要

小李这次想改 design-rules.md,准备走流程
📞 召集让 4 个独立挑战者各自审一遍(这就是 meta-review)
📄 写纪要产出 docs/audits/meta-review-2026-04-26.md
--- covers: - docs/governance/design-rules.md --- ... 审查结论 ...
covers = 这次会议覆盖了哪些文件
✏️ 改文件动 design-rules.md
⏸️ 暂停说话触发保安 1 (Stop)
🚨 保安 1 检查
  • git 看:你确实改了 design-rules.md ✓
  • 纪要 covers 看:有 design-rules.md ✓
  • 纪要后这文件被人 commit 过吗?没有 ✓
✅ 通过,小李可以下班

场景 B:小改动 — 留张纸条

小李只改一个 typo,觉得为这开会太重
✍️ 留纸条在 handoff.md 写
## meta-review: skipped 理由:仅修复 design-rules.md 第 47 行 typo "审查"→"审核",无语义变更
✏️ 改文件
⏸️ 暂停
🚨 保安看到纸条理由非空 ✓
✅ 通过 + 留痕(以后任何人 grep "skipped" 都能看到所有跳过 + 理由)

关键:跳过必须留理由,不能空白。这样事后能审"哪些跳过合理 / 哪些是糊弄"。

场景 C:不开会也不留纸条 — 被拦

小李偷懒,直接改了就想 commit 走人
✏️ 改文件design-rules.md
⏸️ 暂停
🚨 保安 1 检查
  • 有改动?✓
  • 有纸条(skipped)?✗
  • 有有效纪要?✗
❌ 拦!"design-rules.md 没有 meta-review 记录"
😩 小李被迫要么去开会(meta-review),要么去留纸条(skipped + 理由)。**默默溜走的路被堵了**。

两个保安站在哪两扇门?

门 1
Stop
每次 AI 说完话
准备等你输入时
守住"改了但没 commit 就溜走"
门 2
pre-commit
你打 git commit 命令时
守住"改了直接 commit 进 git 历史"

两扇门叠加 = 改动**没法不留痕地溜进 git 历史**。

纪要会"过期"吗?

会。因为审过的文件,后来又被人动过,原纪要可能就不准了。

📅 4/1 开会审了 A.md 和 B.md → 纪要 covers: A, B
📅 4/5 改 A.md → 保安看纪要,A 在覆盖范围 + 4/1 之后没新 commit → ✅ 通过
📅 4/10 别人 commit 了 A.md(改了内容)
📅 4/15 又想改 A.md → 保安看纪要 A 在覆盖,但 4/10 有新 commit → ❌ 纪要对 A 失效,要重审

这比"24 小时窗口"之类的时间限制精确 — 按 git 历史看,文件**实质上动过没**。
